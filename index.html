备忘录

免费好用的电子书下载网

1.ePUBee

http://cn.epubee.com/books/

这个网站应该是全球最大的免费电子书库，超过10万本书籍，50万个文件版本，总能找到你喜欢的适合阅读器的电子书文件。品类齐全，搜索功能强大，图书管理方便。

2.鸠摩搜书

https://www.jiumodiary.com/

一个强大的搜书神站，无论是什么类型的书籍，只要你知道书名，就可以轻松的搜到你想要书籍。这也是我最常用的一个网站。页面简洁到没朋友。

3.Owllook

https://www.owllook.net/

一个免费下载各类小说的搜索引擎， 页面清新简洁，无广告。直接输入书名即可搜索

4.我的小书屋

http://mebook.cc/

提供各类畅销小说，网络小说，以及一些原版书籍下载。不仅如此还支持kindle漫画书下载。

5.西林街

http://www.xilinjie.com/

垂直专注于网盘、视频、文库（文档、古籍、专业书籍、电子书[PDF、ePub、Mobi等格式]）、学术（各种期刊、论文、学报等）和Mooc（在线课程、学习、视频教程）等资源的搜索。

6.书伴（kindle必备）

https://bookfere.com/

书伴，原名“Kindle伴侣”，能让你更加深入地使用手中的Kindle阅读器，让读书成为生命的一部分，让灵魂永远行走在路上。

7.盘搜

http://www.pansoso.com/

一个老牌网盘搜索工具，功能非常强大。非常简洁的界面。每天都有更新，不同达人分享自己的“盘中资源”！盘搜不存储任何网盘内容，无论是工作还是学习都必备。

8.众人搜索网

http://dianzishu.renrensousuo.com/

一个电子书搜索功能的网站，可以同时搜索各类电子书、电子小说等。

gitHub + Hexo 搭建博客

环境：windows 10 + 64 位

1 安装Node.js 及环境配置

  • node -v
  • npm -v

2 安装Git和配置好Git环境

安装成功的象征就是在电脑上任何位置鼠标右键能够出现如下两个选择

注意：一般出于安全考虑，只有在Git Bash Here中才能进行Git的相关操作。如果需要在cmd命令行里调用Git，那么就要配置电脑的环境变量Path，或者在安装的时候选择use Git from the Windows Command Prompt。这个可有可无，影响不大，成功配置的界面如图

3 Github账户注册和新建项目

项目必须要遵守格式：账户名.github.io，并且需要勾选Initialize this repository with a README

在建好的项目右侧有个settings按钮，点击它，向下拉到GitHub Pages，你会看到那边有个网址，访问它，你将会惊奇的发现该项目已经被部署到网络上，能够通过外网来访问它。

4 安装Hexo

在自己认为合适的地方创个文件夹，我是在D盘建了一个blog文件夹 D:\lhyBlog\。然后通过命令行进入到该文件夹里面

  • 输入npm install hexo -g，开始安装Hexo
  • 输入hexo -v，检查hexo是否安装成功
  • 输入hexo init，初始化该文件夹
  • 看到后面的“Start blogging with Hexo！”，成功！
  • 输入npm install，安装所需要的组件
  • 输入hexo g，首次体验Hexo
  • 输入hexo s，开启服务器，访问该网址，正式体验Hexo
    注：假如页面一直无法跳转，那么可能端口被占用了。此时我们ctrl+c停止服务器，接着输入“hexo server -p 端口号”来改变端口号

5 关联Hexo与Github page

首次需要设置Git的user name和email

  • 1 在其文件夹里面鼠标右键
  • 2 点击Git Base Here。这里“feng”可以替换成自己的用户名，邮箱可以替换成自己的邮箱
  • 3 输入cd ~/.ssh，检查是否有.ssh的文件夹
  • 4 输入ls，列出该文件下的内容
  • 5 输入ssh-keygen -t rsa -C “luohaiyan198x@163.com”，连续三个回车，生成密钥，最后得到了两个文件：id_rsa和id_rsa.pub（默认存储路径是：C:\Users\Administrator.ssh）
  • 6 输入eval “$(ssh-agent -s)”，添加密钥到ssh-agent
  • 7 输入ssh-add ~/.ssh/id_rsa，添加生成的SSH key到ssh-agent
  • 8 登录Github，点击头像下的settings，添加ssh
  • 9 新建一个new ssh key，将id_rsa.pub文件里的内容复制上去
  • 10 输入ssh -T git@github.com，测试添加ssh是否成功。如果看到Hi后面是你的用户名，就说明成功了

问题：假如ssh-key配置失败，那么只要以下步骤就能完全解决

  • 1 清除所有的key-pair
    ssh-add -D
    rm -r ~/.ssh

  • 2 删除你在github中的public-key

  • 3 重新生成ssh密钥对
    ssh-keygen -t rsa -C “xxx@xxx.com“

在github上添加公钥public-key:

  • 1 首先在你的终端运行 xclip -sel c ~/.ssh/id_rsa.pub将公钥内容复制到剪切板
  • 2 在github上添加公钥时，直接复制即可
  • 3 保存
